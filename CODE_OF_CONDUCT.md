# Code of conduct

The KiCI project adopts the [Contributor Covenant, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) as its code of conduct in full.

The full text of the covenant is the authoritative source. The summary below is informational only.

## What this means in practice

- We're a welcoming community. Be kind, be patient, assume good faith.
- Disagree about technical decisions on the merits, not on the person making them.
- Harassment, discrimination, and personal attacks have no place here — they are grounds for removal from the project's spaces (issues, PRs, discussions, related chat).
- The standards apply in every space the KiCI project occupies, and when an individual is representing the project in public.

## Reporting

Conduct concerns are handled in private. Please file a [private GitHub Security Advisory](https://github.com/kici-dev/kici-public/security/advisories/new) titled "conduct report" and include:

- What happened, when, and where (link to the issue / PR / message thread if applicable).
- Who was involved (handles).
- Any context the reporter feels is relevant.

We chose the Security Advisory channel because GitHub keeps it private to the reporter and the maintainers, with no public visibility. We will acknowledge within 7 business days and follow up with the outcome.

## Enforcement

Maintainers will follow the [Contributor Covenant enforcement ladder](https://www.contributor-covenant.org/version/2/1/code_of_conduct/#enforcement-guidelines) (correction → warning → temporary ban → permanent ban) and document each enforcement decision internally. The covenant text linked above describes the four levels and the criteria for each.
